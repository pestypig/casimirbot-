You just need to treat the Phase‐Diagram’s two sliders (“Tile area” and “Ship radius”) exactly the same way you already do for the other preset parameters. In practice that means:

1) Lifting the “tileArea” and “shipRadius” values into your shared state  
2) Having your “Apply Needle-Hull Preset” button call the same setter functions that those sliders use  
3) Binding your Phase tab’s sliders to that shared state so they move in lock‐step  

Here’s how it breaks down in both React (client-side) and Streamlit (Python) flavors:

––––––––––––––––––––––––––––––––––––––––  
A) React/Redux (or useContext/useState) approach  
––––––––––––––––––––––––––––––––––––––––  
// 1) In your top-level component (e.g. App.jsx):

```jsx
import { useState } from 'react';
import PhaseTab from './components/PhaseTab';
import SimulationPanel from './components/SimulationPanel';

function App() {
  // Lifted state
  const [tileArea,   setTileArea]   = useState(2500);  // cm²
  const [shipRadius, setShipRadius] = useState(5.0);   // m

  // Your existing “Apply Needle-Hull Preset”
  function applyNeedleHullPreset() {
    // Other presets you already do…
    setTileArea(2500);
    setShipRadius(5.0);
    // plus Q, gamma, duty, etc.
  }

  return (
    <>
      <SimulationPanel
        onApplyPreset={applyNeedleHullPreset}
        // …other props
      />
      <Tabs>
        <Tab label="Results">…</Tab>
        <Tab label="Phase">
          <PhaseTab
            tileArea={tileArea}
            shipRadius={shipRadius}
            onTileAreaChange={setTileArea}
            onShipRadiusChange={setShipRadius}
          />
        </Tab>
        …  
      </Tabs>
    </>
  );
}
```

// 2) In `PhaseTab.jsx`, make your sliders **controlled** by those props:

```jsx
import PlotlyHeatmap from './PlotlyHeatmap';

export default function PhaseTab({
  tileArea, shipRadius,
  onTileAreaChange, onShipRadiusChange
}) {
  return (
    <div>
      <label>Tile area (cm²)</label>
      <input
        type="range" min={100} max={10000}
        value={tileArea}
        onChange={e => onTileAreaChange(+e.target.value)}
      />
      <span>{tileArea}</span>

      <label>Ship radius (m)</label>
      <input
        type="range" min={1} max={30}
        value={shipRadius}
        onChange={e => onShipRadiusChange(+e.target.value)}
      />
      <span>{shipRadius.toFixed(1)}</span>

      <PlotlyHeatmap tileArea={tileArea} shipRadius={shipRadius} />
    </div>
  );
}
```

Now when the user clicks **Apply Needle-Hull Preset**, it calls `setTileArea(2500)` & `setShipRadius(5.0)`, which immediately updates the sliders in the Phase tab and reruns the heat‐map with those “golden” values.

––––––––––––––––––––––––––––––––––––––––  
B) Streamlit approach  
––––––––––––––––––––––––––––––––––––––––  
Streamlit offers `st.session_state` so you can do the same:

```python
# In your main app.py (or wherever you wire the “Apply Preset” button)
if st.button("Apply Needle-Hull Preset"):
    st.session_state.tile_area   = 2500
    st.session_state.ship_radius = 5.0

# Then in ui/phase_tab.py:
import streamlit as st
from sim_core.phase import viability, np, params

# Initialize defaults if not present
if "tile_area" not in st.session_state:
    st.session_state.tile_area   = 2500
if "ship_radius" not in st.session_state:
    st.session_state.ship_radius = 5.0

tile_slider = st.slider(
    "Tile area (cm²)",
    min_value=100,
    max_value=10000,
    value=st.session_state.tile_area,
    key="tile_area"
)
ship_slider = st.slider(
    "Ship inner-hull radius (m)",
    min_value=1.0,
    max_value=30.0,
    value=st.session_state.ship_radius,
    key="ship_radius"
)

# Build and render the heatmap as before, using tile_slider and ship_slider
…
```

Because the **preset** button writes into `st.session_state.tile_area` & `.ship_radius`, Streamlit will automatically update those sliders (and your `Phase` tab) on the next rerun—no extra wiring needed.

––––––––––––––––––––––––––––––––––––––––  
Result  
––––––––––––––––––––––––––––––––––––––––  
Once you hook your preset action up to set the *same* state variables that the Phase‐Diagram sliders read from, clicking **Apply Needle-Hull Preset** will snap your heatmap cursor to (2500 cm², 5 m) and instantly reveal that sweet teal “viable” region.
