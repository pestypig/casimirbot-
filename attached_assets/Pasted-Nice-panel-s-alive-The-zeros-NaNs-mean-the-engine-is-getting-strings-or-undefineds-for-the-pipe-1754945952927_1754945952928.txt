Nice—panel’s alive! The zeros/NaNs mean the engine is getting **strings or undefineds** for the pipeline numbers, so the proof math multiplies to 0 and duty shows `NaN%`.

### What the readout is telling us

* `mode: hover` even after you switch → the engine probably isn’t getting `currentMode`, or it’s not merged on change.
* `β_inst/β_avg/β_net = 0` → at least one of `{gammaGeo, Qburst|cavityQ, deltaAOverA}` is `undefined` or a **string** (e.g., `"1.0e+9"`, `"26.0"`).
* `duty: NaN%` → `dutyCycle` is undefined or a string (like `"14.0%"`).

### Fast fix in `WarpVisualizer.tsx`

Coerce everything to numbers, normalize duty to a **fraction**, and pass the exact names the engine expects.

```ts
// helper
const num = (v: any, def = 0) => {
  const n = Number(v);
  return Number.isFinite(n) ? n : def;
};

const mode = parameters.currentMode || 'hover';

// duty as fraction [0..1]
const dutyFrac = (() => {
  const d = String(parameters.dutyCycle ?? '').replace('%','').trim();
  const n = Number(d);
  if (!Number.isFinite(n)) return 0;
  return n > 1 ? n/100 : n;
})();

engineRef.current.updateUniforms({
  // physics (all numbers)
  dutyCycle:   dutyFrac,
  gammaGeo:    num(parameters.g_y),
  Qburst:      num(parameters.cavityQ),            // maps to engine Q
  deltaAOverA: num(parameters.qSpoilingFactor, 1), // Δa/a
  gammaVdB:    num(parameters.gammaVanDenBroeck, 1),

  sagDepth_nm: num(parameters.sagDepth_nm),
  powerAvg_MW: num(parameters.powerAvg_MW),
  exoticMass_kg:num(parameters.exoticMass_kg),
  tsRatio:      num(parameters.tsRatio, 4100),

  // mode/visual
  currentMode: mode,
  sectorCount: num(parameters.sectorStrobing, 1),  // was sectorStrobing
  phaseSplit:  mode === 'cruise' ? 0.65 : mode === 'hover' ? 0.50 : 0.70,
  viewAvg:     1.0,
  betaGain:    1e-10
});

// (optional) debug
console.table(engineRef.current.uniforms);
```
