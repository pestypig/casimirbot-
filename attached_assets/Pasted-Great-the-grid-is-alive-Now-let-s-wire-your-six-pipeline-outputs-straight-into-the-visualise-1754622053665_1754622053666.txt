## 2 · Live-hook the dashboard or pipeline JSON

```html
<!-- simple HTML controls for quick tests -->

  engine.updateUniforms({
    dutyCycle: +duty.value,
```

*Drag a slider* → `updateUniforms` fires → next frame calls `_warpGridVertices` and the grid morphs before your eyes.

If your “energy pipeline solution” arrives over WebSocket/REST, call `engine.updateUniforms(freshNumbers)` inside the handler instead.

## 3 · (Optional) make curvature respond to **negative energy density**

Right now the grid only bends laterally & vertically by `β`.
To visualise the *volume integral* (exotic mass) add a depression proportional to **–ρ**:

```js
// bottom of _warpGridVertices()
const rho = -beta*beta * bubbleParams.cavityQ;   // toy ρ≈–β²Q
const extraDip = rho * 5e-3;                     // scale for visibility
vtx[i+1] += extraDip;
```

When **Exotic Mass** climbs, the whole mesh sinks deeper, showing how much spacetime you “bought” with the energy pipeline.

## 4 · Colour-code the grid itself (quick heat-map)

Change the grid fragment shader to tint by vertical stretch:

```glsl
float h = abs(gl_Position.y);              // 0 = flat, 1 = deep well
frag = vec4(mix(vec3(0.2,0.8,1.0),         // cyan
                vec3(1.0,0.3,0.1), h*4.0), 1.0);
```

Blue lines stay near equilibrium; orange/red show regions where the pipeline is forcing large curvature.
