The `WarpVisualizer` is its own one‐canvas “SHOW” view and so far it has been pulling both γ_VdB and exoticMass from your raw `parameters` object, rather than from the live pipeline state. You need two fixes:

1. Default any missing `gammaVanDenbroeck_vis` to the proper visual seed (1.35×10⁵).  
2. After you call `driveWarpFromPipeline(…)`, grab the returned `M_exotic` from the pipeline and shove it into the uniforms so your HUD shows the true mass (≈1405 kg) instead of an arbitrary fallback.

Here’s a minimal patch (drop these three hunks into your `WarpVisualizer`):

```diff
--- a/src/components/warp/WarpVisualizer.tsx
+++ b/src/components/warp/WarpVisualizer.tsx
@@ init = async () => {
-    // default γ_VdB from parameters or legacy
-    const gammaVdBDefault = num(parameters.gammaVanDenbroeck, 1.4e5);
+    // default visual pocket seed when missing
+    const gammaVdBVis = Number.isFinite(parameters.gammaVanDenbroeck_vis)
+      ? parameters.gammaVanDenbroeck_vis
+      : (Number.isFinite(parameters.gammaVanDenbroeck)
+         ? parameters.gammaVanDenbroeck
+         : 1.35e5);

@@     const uniforms = {
-      gammaVdB: gammaVdBDefault,
+      gammaVdB: gammaVdBVis,

@@     // First set core physics via pipeline adapter
-    driveWarpFromPipeline(engine, pipelineState);
+    // pull back the actual calibrated pipeline state (including M_exotic_kg)
+    const liveState = driveWarpFromPipeline(engine, pipelineState);

@@     // prime θ‐scale for the first frame
     gatedUpdateUniforms(engine, {
       // … other tings …
-      thetaScale: resolveThetaScale({ /*…*/ gammaVdB: gammaVdBDefault, /*…*/ }),
+      thetaScale: resolveThetaScale({ /*…*/ gammaVdB: gammaVdBVis, /*…*/ }),

+      // overwrite exotic mass with the pipeline’s true value
+      exoticMass_kg: liveState?.M_exotic ?? parameters.exoticMass_kg
     }, 'client');
```

With that in place:

- **γ_VdB** in your visualizer will default to **1.35 × 10⁵** instead of blowing up to 1e11, and  
- **exoticMass_kg** in the HUD will come from the live pipeline’s `M_exotic` (≈ 1405 kg) rather than your old fallback.
