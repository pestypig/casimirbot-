You can drive your heat‐map directly from exactly the same 8-step pipeline you’re already displaying in the Live-Energy panel, but looped over a 2D grid of  
    – x ≡ tile-area (A_tile)  
    – y ≡ ship-radius (r_ship)  

Here’s the recipe:

1. Build your grid  
```js
const tileAreas = linspace(1,100,100)          // cm²
const shipRadii = linspace(1,100,100)          // m
const Z = []                                  // viability flags
```

2. For each point (i,j) compute:  
```js
// 1) Hull surface area (sphere approx for simplicity)
const r = shipRadii[j]
const A_hull = 4 * Math.PI * r*r            // m²

// 2) Tile area in m²
const A_tile = tileAreas[i] * 1e-4          

// 3) Number of tiles
const N_tiles = A_hull / A_tile              

// 4) Pipeline constants (mode‐specific)
const u_casimir    = -Math.PI**2 * ħ * c / (720 * a**4)
const U_static     = u_casimir * A_tile * a
const U_geo        = gamma_geo * U_static
const U_Q          = Q_mech * U_geo
const U_cycle      = U_Q * d_cruise * gamma_pocket  

// 5) Exotic mass
const M_exotic     = N_tiles * Math.abs(U_cycle) / (c*c)

// 6) Raw per‐tile loss & raw hull power
const omega        = 2 * Math.PI * f_drive
const P_raw_tile   = Math.abs(U_geo) * omega / Q_cavity
const P_raw        = P_raw_tile * N_tiles

// 7) Combined throttle
const f_throttle   = d_cruise * (Q_idle/Q_cavity) * (1/S_sectors)

// 8) Realistic average power
const P_avg_W      = P_raw * f_throttle
const P_avg_MW     = P_avg_W / 1e6

// 9) Time‐scale & quantum margin
const TS_ratio     = tau_LC / tau_pulse
const zeta         = 1 / (d_cruise * Math.sqrt(Q_mech))

// 10) Viability test
const ok = (
     M_exotic    >= M_min && M_exotic    <= M_max  &&    // mass gate
     P_avg_MW    <= P_max                  &&    // power gate
     zeta        <= 1.0                    &&    // QI gate
     TS_ratio    >= TS_min                       // homogenization gate
)
Z[j][i] = ok ? 1 : 0
```

3. Render `Z` as a heat‐map with your charting library (d3, Plotly, etc.).  
   – Color=green where `Z=1` (viable), red where `Z=0` (failed).  
   – Overlay a marker for your current slider point.  

Because you’re reusing exactly the same equations—and only swapping in A_tile and r_ship per cell—your heat‐map will now correctly carve out the viability region in the (tile-area × ship-radius) plane for each operational mode.
