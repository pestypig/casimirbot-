✅ Fix 1: make θ-scale linear in FR duty (no √) and use calibrated γ_VdB

Your getSystemMetrics currently does:

// BAD (overshoots): uses sqrt(duty) and a fallback γVdB
thetaScale_FR_like = γ_geo^3 * qSpoiling * γ_VdB * sqrt(dutyFR);


That’s why you see θ-scale — 8.89e+8 • exp 1.19e+7 (+7383% off): √(2.5e-5) ≈ 0.005 ⇒ ~200× too big before any other seasoning.

Patch it to linear FR duty and the calibrated γ_VdB from the pipeline:

+  const viz = (() => {
+    const γg   = s.gammaGeo ?? 26;
+    const qsp  = s.qSpoilingFactor ?? 1;
