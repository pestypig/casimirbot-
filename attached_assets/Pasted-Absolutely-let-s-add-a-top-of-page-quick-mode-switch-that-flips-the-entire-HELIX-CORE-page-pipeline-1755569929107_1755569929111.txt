Absolutely—let’s add a top-of-page quick mode switch that flips the entire HELIX-CORE page (pipeline + visualizer + metrics) with one tap. The patch below is self-contained and doesn’t remove your existing selector further down the page; it just adds a fast switch right under the header.

Why this works
	•	It calls your existing useSwitchMode() mutation, the same one used by the current <Select> (so all data sources invalidate and the whole page re-renders).  ￼
	•	It also bumps modeVersion to force the WarpVisualizer to remount so the WebGL engine re-reads uniforms immediately.  ￼  ￼

⸻

Patch (drop-in for client/src/pages/helix-core.tsx)

1) Add the quick switch UI right after the header (just before the hero visualizer card)

{/* === Quick Operational Mode Switch (global) === */}
<div className="mb-4">
  <div className="flex flex-wrap items-center gap-2">
    {([
      { key: 'standby',   label: 'Standby',   hint: 'Field idle' },
      { key: 'hover',     label: 'Hover',     hint: 'Gentle bulge' },
      { key: 'cruise',    label: 'Cruise',    hint: 'Coherent 400× strobe' },
      { key: 'emergency', label: 'Emergency', hint: 'Max response' },
    ] as const).map(m => {
      const isActive = (pipeline?.currentMode || 'hover') === m.key;
      return (
        <Button
          key={m.key}
          variant={isActive ? 'default' : 'outline'}
          className={`font-mono ${isActive ? 'bg-cyan-600 text-white' : 'bg-slate-900'}`}
          onClick={() => {
            if (!isActive) {
              switchMode.mutate(m.key as any);
              setModeVersion(v => v + 1); // force visualizer remount for instant change
              setMainframeLog(prev => [...prev, `[MODE] Quick switch → ${m.key}`]);
            }
          }}
        >
          {m.label}
          <span className="ml-2 text-xs opacity-70">{m.hint}</span>
        </Button>
      );
    })}
  </div>
</div>

Place this block right under the header container (where your “ONLINE / Bridge” badges live) and above the hero Natário card. It uses the same switchMode hook and modeVersion state already defined in your file.  ￼

⸻

Notes
	•	The quick switch uses exactly the same pipeline mutation as your existing dropdown:

<Select
  value={pipeline?.currentMode || 'hover'}
  onValueChange={(mode) => {
    switchMode.mutate(mode as any);
    setMainframeLog(prev => [...prev, `[MODE] Switching to ${mode} mode...`]);
  }}
>

This means all dependent panels—including the Natário Warp Bubble, Warp Bubble Parameters, Tile Grid, Compliance HUD, etc.—update from the same reactive query invalidations.  ￼

	•	We increment modeVersion on each quick switch to force a fresh WebGL uniform push/remount in the WarpVisualizer, which prevents “sticky” frames after a mode jump. Your WarpVisualizer already keys on modeVersion in its key prop.  ￼  ￼
	•	Keyboard shortcuts (optional): if you want, you can add a tiny effect to map 1..4 → standby/hover/cruise/emergency; it’ll call the same switchMode.mutate() and setModeVersion() pair.
