# NHM2 Tile-Effective Full-Tensor Source Claim Safety

This patch adds source-side tile-effective full-tensor counterpart evidence for the frozen NHM2 validation-hardening ledger. It does not validate NHM2, certify transport, prove a physical GR-plus-quantum mechanism, or permit ambient faster-than-light language.

The permitted claim is narrow:

```text
NHM2 emits source-side full-tensor tile-effective counterpart candidates so the frozen blocker ledger can distinguish missing counterpart authority from source-to-geometry residuals, QEI linkage gaps, and conservation gaps.
```

## Source-Side Requirement

The tile-effective full-tensor source artifact is only useful if it is not a metric echo. A source-side candidate must declare:

```text
sourceSideOnly = true
notDerivedFromMetricRequiredTensor = true
metricRequiredInputRefs = []
```

Any metric-required tensor ref in the source path is a blocker. Copying `G_ab[g_NHM2] / 8pi` into the tile side proves only that the metric-required tensor can be restated; it does not establish a supporting source.

## Tensor Authority

Promotion-sensitive source closure requires full tensor or explicit symmetric full-tensor authority. Diagonal reduced-order and proxy tensors remain diagnostic evidence only. They can identify blocker locations, but they cannot retire source-legitimacy blockers.

## Same-Basis Surface

Each controlled region must carry the same chart, units, region mask, aggregation mode, normalization basis, and sample count used for comparison against the metric-required tensor:

```text
global
hull

exterior_shell
```

Missing regions or unknown basis metadata keep the ledger in review or fail.

## QEI And Conservation

QEI/QFT papers provide guardrails, not proof that the NHM2 tile source exists. A source-side tensor candidate still needs a worldline-based QEI dossier and conservation diagnostics before physical-mechanism language can be considered.

`validationClaimAllowed`, `physicalMechanismClaimAllowed`, and `promotionAllowed` remain false.
