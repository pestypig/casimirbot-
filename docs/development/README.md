# Current Development Notes

Use this folder for docs that still describe the active implementation path.
These are working notes, contracts, runbooks, and plans that a developer should
be able to trust when making a current patch.

Good candidates:

- active architecture contracts
- current runbooks and operational procedures
- implementation plans that still match the codebase
- API, adapter, and viewer behavior notes
- current Helix Ask, mission-control, or docs-viewer contracts

Do not put research papers here. Research-grade documents with citations,
equations, sidecars, or formal claim boundaries belong in `../research/`.
